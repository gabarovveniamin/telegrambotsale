# 📓 Work Log: Telegram Discount Monitor Bot (Kazakhstan)

## 🏗️ Описание проекта на текущий момент
Проект представляет собой **высоконагруженную систему мониторинга скидок** для крупнейших E-commerce площадок Казахстана, интегрированную в Telegram. Бот не просто собирает данные, а выступает в роли полноценного агрегатора выгодных предложений с элементами персонализации и монетизации.

### Основной функционал:
- **Агрегация скидок**: Автоматический сбор данных из Technodom, Sulpak, Mechta, Alser и Kaspi.
- **Персонализация**: Пользователи могут настраивать "порог срабатывания" (например, уведомлять только о скидках от 30%).
- **Точечная следилка**: Возможность добавить конкретную ссылку на товар (Kaspi/Sulpak/Mechta) для ежечасного мониторинга цены.
- **Монетизация (Premium)**: Система платных подписок через **Telegram Stars**.
  - Актуальный ценник: **150 Stars / месяц**.
  - Premium открывает доступ к автоматическим уведомлениям и точечной следилке.
- **Реферальная программа**: Система привлечения новых пользователей с бонусами (7 дней Premium за каждого приглашенного друга, совершившего покупку).
- **Админ-панель**: Полноценный центр управления внутри бота (статистика, управление пользователями, рассылка, ручная выдача Premium, запуск парсеров).

## 🔍 Технические тонкости и архитектура
Проект построен с учетом современных требований к безопасности и производительности:

1.  **Обход анти-бот систем (WAF/Cloudflare)**:
    - Использование библиотеки `curl_cffi` с имперсонацией браузера Chrome. Это позволяет боту успешно парсить сайты, которые блокируют стандартные Python-библиотеки (`requests`, `httpx`).
2.  **Асинхронность**:
    - Полностью асинхронный стек (`aiogram 3.x`, `asyncpg`, `AsyncSession`). Бот способен обрабатывать тысячи сообщений в секунду, не блокируя основной поток выполнения.
3.  **База данных (PostgreSQL)**:
    - Проведен переход с SQLite на PostgreSQL для обеспечения консистентности данных и возможности масштабирования.
    - Использование пулов соединений (`asyncpg.create_pool`) для эффективной работы под нагрузкой.
4.  **Безопасность**:
    - Вся конфигурация (токены, пароли БД) вынесена в `.env` файлы.
    - История Git очищена от старых чувствительных данных.
5.  **Надежность рассылки**:
    - Реализованы алгоритмы соблюдения лимитов Telegram API (Rate Limiting) для предотвращения бана бота при массовых рассылках уведомлений.
